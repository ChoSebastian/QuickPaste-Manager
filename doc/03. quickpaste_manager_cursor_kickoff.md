# Cursor 개발 착수본 - QuickPaste Manager

이 문서는 Cursor에 바로 제공할 개발 착수용 지시서입니다.  
목표는 **QuickPaste Manager 프로젝트를 로컬 홈폴더 기준으로 구조화하고, Git 초기화와 GitHub 연동 준비까지 포함한 실제 개발 시작 상태**를 만드는 것입니다.

---

## 1. 프로젝트 기본 정보

- 프로젝트명: QuickPaste Manager
- 로컬 홈폴더: `D:\Smit_Dev\Python\QuickPaste Manager`
- 주 개발 언어: Python 3.12
- GUI: PySide6
- 데이터 저장: SQLite
- 실행 환경: Windows 10/11 64-bit
- 패키징 목표: PyInstaller + Inno Setup

---

## 2. Cursor에게 기대하는 역할

당신은 로컬 개발 보조 에이전트이다.  
이 프로젝트에서 해야 할 일은 다음과 같다.

1. 홈폴더를 기준으로 프로젝트 구조를 정리한다.
2. Python 데스크톱 앱의 초기 뼈대를 생성한다.
3. Git 저장소 초기화에 필요한 파일을 만든다.
4. GitHub 연동 준비에 필요한 README, `.gitignore`, 초기 커밋 가이드를 작성한다.
5. PRD 범위를 넘는 기능은 추가하지 않는다.
6. 모호한 부분은 TODO로 남기되, 파일 구조는 완성도 있게 잡는다.

중요:
- 실제 GitHub 원격 저장소 생성이나 로그인은 사용자 인증이 필요할 수 있으므로, 가능한 부분까지만 자동화하고 필요한 항목은 명확히 안내한다.
- 실제 OS 저수준 입력 주입은 구조와 인터페이스를 먼저 만들고 위험한 부분은 TODO로 둔다.
- 범위를 과도하게 넓히지 않는다.

## 5. Git 및 GitHub 준비 지시

### 5.1 Git 초기화
가능하면 아래를 수행하라.
- `git init`
- 기본 브랜치 `main`
- `.gitignore` 생성
- 초기 커밋 준비 상태 만들기

### 5.2 .gitignore에 포함할 항목
반드시 아래 범주를 포함하라.
- `__pycache__/`
- `*.pyc`
- `.venv/`
- `venv/`
- `.idea/`
- `.cursor/` 중 로컬 개인 설정 파일
- `dist/`
- `build/`
- `*.spec`
- `logs/`
- `backups/`
- `exports/`
- `.pytest_cache/`
- `.mypy_cache/`
- `.coverage`
- `.DS_Store`

### 5.3 GitHub 연동 준비
실제 원격 저장소 생성과 인증은 사용자 개입이 필요할 수 있다.  
따라서 다음까지 수행하라.
- README에 GitHub 연결 절차 작성
- 원격 저장소 연결 명령 예시 작성
- 첫 push 명령 예시 작성

예시:
```bash
git remote add origin <GITHUB_REPO_URL>
git add .
git commit -m "Initial project scaffold"
git push -u origin main
```

중요:
- 사용자의 인증 정보나 토큰을 임의로 만들지 말 것
- 원격 저장소 URL이 없으면 placeholder를 둘 것

## 7. 실패 처리 구현 원칙

제품 요구사항상 실패 시 경고 후 재실행이 중요하다.  
하지만 착수 단계에서는 아래 방식으로 구현하라.

1. 예외를 로깅한다.
2. 사용자 경고 메시지 표시 함수를 둔다.
3. 재시작 로직은 `restart_manager.py`로 분리한다.
4. 실제 즉시 재시작은 안전한 조건에서만 수행하고, 위험한 경우 TODO로 둔다.
5. 이미지 붙여넣기 실패 시 폴더 열기 유틸 함수를 만든다.

주의:
- 재시작은 무한 루프가 되지 않게 보호 장치를 둔다.
- DB 쓰기 중 강제 종료는 피한다.
