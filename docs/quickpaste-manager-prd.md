# QuickPaste Manager 최종 PRD

- 문서 버전: 1.0
- 작성 목적: Windows용 완성형 상용구 관리 유틸리티 개발 착수
- 제품명: QuickPaste Manager
- 프로젝트 홈폴더: `D:\Smit_Dev\Python\QuickPaste Manager`
- 권장 개발 도구: Cursor
- 참고 원문: 사용자 제공 기획서 기반 재구성

---

## 1. 제품 정의

QuickPaste Manager는 Windows 환경에서 자주 사용하는 텍스트와 이미지 자산을 카테고리별로 저장하고, 전역 단축키 또는 선택적 마우스 트리거로 커서 근처 팝업을 호출하여 즉시 붙여넣을 수 있게 해주는 상주형 생산성 유틸리티이다.

이 제품은 Windows 우클릭 메뉴를 수정하는 방식이 아니라, **글로벌 호출 + 자체 팝업 GUI + 클립보드 등록 + 자동 붙여넣기** 방식으로 동작한다.

---

## 2. 목표

### 2.1 제품 목표
반복 입력 작업을 빠르게 줄이고, 사용자가 어떤 프로그램에서 작업 중이든 가능한 한 짧은 동선으로 텍스트/이미지 상용구를 삽입할 수 있게 한다.

### 2.2 비즈니스 목표
- 개인 또는 소규모 실무 환경에서 바로 설치해 쓸 수 있는 완성형 Windows 유틸리티를 만든다.
- 별도 서버 없이 로컬 실행만으로 제품이 성립해야 한다.
- 설치, 실행, 상주, 호출, 관리, 붙여넣기, 내보내기/불러오기까지 하나의 제품 경험으로 완결되어야 한다.

### 2.3 이번 버전 원칙
- MVP가 아니라 배포 가능한 완성형 1차 제품을 목표로 한다.
- 다만 불필요한 확장은 하지 않는다.
- AI 기능은 넣지 않는다.
- 팀 기능은 넣지 않는다.
- 클라우드 동기화는 넣지 않는다.
- 보안/암호화는 넣지 않는다.
- Windows 로그인 사용자라면 누구나 사용 가능한 로컬 유틸리티로 본다.

## 3. 타깃 사용자

### 3.1 1차 타깃 사용자
사무직 직원

### 3.2 사용자 정의
- Windows PC에서 업무를 수행한다.
- 반복적으로 같은 문구, 계좌번호, 응답 문장, 이메일 템플릿, 서명, 로고, 도장 이미지를 붙여넣는다.
- 프로그램을 오가며 찾고 복사하는 과정을 번거롭게 느낀다.
- 고급 권한 관리나 보안 기능보다 빠른 사용성과 편의성을 우선한다.

### 3.3 사용자 가정
- Windows에 로그인한 사용자라면 누구나 앱을 사용할 수 있다.
- 별도 관리자 승인 체계나 권한 분리 체계는 고려하지 않는다.
- 프로그램 실패 시 경고 후 재실행되는 흐름도 수용 가능하다.

## 4. 핵심 문제

사용자는 자주 쓰는 텍스트나 이미지를 반복적으로 찾고 복사하고 붙여넣느라 업무 흐름이 끊긴다.  
특히 문구가 여러 개일수록 메모장, 문서, 카톡, 이메일, 파일 탐색기를 오가며 찾는 비용이 누적된다.

QuickPaste Manager는 이 과정을 다음 흐름으로 단축한다.

`호출 → 검색/선택 → 즉시 붙여넣기`

---

## 5. 핵심 가치

1. 작업 중인 앱을 벗어나지 않고 바로 삽입할 수 있다.
2. 텍스트뿐 아니라 이미지 자산도 같은 방식으로 다룰 수 있다.
3. 검색과 카테고리로 원하는 항목을 빠르게 찾을 수 있다.
4. 시스템 트레이 상주형이라 하루 종일 사용하기 쉽다.
5. 서버 없이 로컬만으로 완결된다.

## 7. 사용 시나리오

### 시나리오 A: 고객 응대 문구 삽입
1. 사용자가 메신저나 브라우저 입력창에서 작업 중이다.
2. `Ctrl + Shift + V`를 누른다.
3. 커서 옆에 QuickPaste 팝업이 뜬다.
4. 검색창에 일부 키워드를 입력하거나 카테고리에서 문구를 찾는다.
5. 문구를 선택한다.
6. 프로그램이 클립보드에 내용을 넣고 자동 붙여넣기를 시도한다.
7. 성공 시 팝업이 닫히고 사용 기록이 반영된다.

### 시나리오 B: 서명/로고 이미지 삽입
1. 사용자가 문서 편집기 또는 메신저에서 작업 중이다.
2. 팝업을 띄운다.
3. 이미지 항목을 선택한다.
4. 프로그램이 이미지 클립보드 등록 후 자동 붙여넣기를 시도한다.
5. 실패하면 이미지 저장 경로가 들어 있는 폴더를 열고, 로그를 남기고, 프로그램을 종료 후 재실행한다.

### 시나리오 C: 상용구 관리
1. 사용자가 시스템 트레이에서 관리 화면을 연다.
2. 카테고리를 선택한다.
3. 텍스트 또는 이미지를 새로 추가한다.
4. 제목/태그/카테고리를 지정한다.
5. 저장 후 즉시 팝업에서 사용할 수 있다.

## 8. 기능 요구사항

## 8.1 설치 및 최초 실행
- 설치 파일 하나로 설치 가능해야 한다.
- 설치 후 최초 실행 시 기본 설정 마법사를 제공한다.
- 최초 설정 항목:
  - 호출 단축키
  - 마우스 휠 클릭 사용 여부
  - 시작 프로그램 등록 여부
  - 기본 데이터 저장 위치
  - 샘플 상용구 생성 여부
- 완료 후 시스템 트레이에 상주해야 한다.

## 8.2 시스템 트레이
트레이 메뉴는 다음을 제공해야 한다.
- 팝업 열기
- 상용구 관리
- 환경설정
- 내보내기
- 불러오기
- 도움말
- 종료

## 8.3 글로벌 호출
- 기본 호출 단축키는 `Ctrl + Shift + V`
- 사용자 지정 가능
- 선택적으로 마우스 휠 클릭 호출 가능
- 호출 시 현재 마우스 좌표 기준 팝업 표시
- 화면 끝에서는 반대 방향으로 보정
- 다중 모니터 대응
- DPI 배율 대응

## 8.4 팝업 UI
팝업에는 다음이 포함되어야 한다.
- 최상단 검색창
- 사용 빈도 기반 Top 5
- 1단 카테고리 목록
- 선택 항목 목록
- 빠른 액션 버튼

기본 요구:
- 팝업 크기 기본 360 x 520px
- 커서 우측 12px 위치
- ESC, 외부 클릭, 붙여넣기 완료 시 닫힘
- 텍스트는 1~2줄 미리보기
- 이미지는 썸네일 표시

## 8.5 카테고리 정책
- 카테고리는 1단 구조만 사용
- 최대 10개까지 사용자가 생성 가능
- 최초 실행 시 기본 카테고리 5개 생성
- 각 카테고리에는 약 100개 수준의 항목 수용을 목표로 한다.

## 8.6 상용구 데이터 유형
지원 유형:
- 일반 텍스트
- 긴 텍스트
- 코드 스니펫
- 이미지
- 제목 + 설명 + 본문 + 이미지 참조를 가진 복합 항목

## 8.7 상용구 추가/수정/삭제
추가 방법:
- 직접 텍스트 입력
- 현재 클립보드 내용에서 추가
- 이미지 파일 선택
- 현재 클립보드 이미지 저장

수정 가능 항목:
- 제목
- 본문
- 카테고리
- 태그
- 이미지
- 고정 여부

삭제 정책:
- 즉시 영구 삭제가 아니라 휴지통 방식
- 복구 가능
- 환경설정에서 휴지통 비우기 제공

## 8.8 검색
- 팝업 최상단 검색창 제공
- 제목, 태그, 본문 일부 기준 검색
- 검색은 카테고리 탐색보다 빠른 접근 수단으로 동작해야 한다.

## 8.9 Top 5
- 단순 사용 횟수만이 아니라 최근 사용성과 고정 여부를 반영한다.
- 기본 점수 정책:
  - use_count 반영
  - 최근 24시간 가산
  - 최근 7일 가산
  - pinned 가산
  - 비활성/삭제 항목 제외

## 8.10 붙여넣기
### 텍스트
- 선택 항목을 클립보드 Unicode 텍스트로 등록
- 팝업 닫기
- 짧은 지연 후 `Ctrl + V` 자동 입력
- 사용 횟수 및 마지막 사용 시각 갱신

### 이미지
- 가능한 경우 표준 이미지 클립보드 형식으로 등록
- 자동 붙여넣기 시도
- 지원 앱에서는 이미지 삽입
- 실패 시:
  1. 경고 표시
  2. 이미지 파일 저장 경로 폴더 열기
  3. 로그 기록
  4. 프로그램 종료 후 재실행

### 이미지 정책
- 이미지 기능은 우선 구현 대상이지만,
- 개발 복잡도나 오류율이 과도하면 최종 릴리스 범위를 텍스트 중심으로 제한할 수 있다.
- 즉, 텍스트 붙여넣기는 절대 필수, 이미지는 조건부 필수로 본다.

## 8.11 환경설정
설정 항목:
- 호출 단축키
- 마우스 휠 클릭 사용 여부
- 시작 프로그램 등록
- 팝업 위치 보정
- 테마
- 글자 크기
- 자동 붙여넣기 지연
- 클립보드 복원 여부
- 이미지 붙여넣기 방식
- 자동 백업 주기
- 내보내기 / 불러오기

## 8.12 Import / Export
### Export
ZIP 패키지로 내보내기
포함 항목:
- manifest.json
- snippets.json
- categories.json
- settings.json
- assets 폴더

### Import
- 동일 제목 존재 시 사용자 선택
- 동일 이미지 존재 시 중복 방지
- 없는 카테고리는 자동 생성
- 손상된 ZIP은 불러오기 중단
- 기존 데이터는 유지

### 공유 방식
- 원하는 사용자에게 내보내기 파일을 전달하고,
- 상대방은 불러오기로 읽는 방식까지만 지원한다.

## 8.13 백업
- 자동 백업 주기 설정 가능
- 최근 백업 복원 기능 제공
- 앱 비정상 종료 후 복구 안내 가능

## 9. 실패 처리 정책

사용자 입력 기준으로 다음 정책을 적용한다.

### 9.1 공통 원칙
- 실패 시 경고를 띄운다.
- 로그를 남긴다.
- 프로그램을 종료하고 재실행한다.
- 재실행 후에도 안 되면 사용자가 수동으로 처리한다.

### 9.2 실패 케이스
- 단축키 등록 실패
- 클립보드 접근 실패
- 자동 붙여넣기 실패
- 이미지 로드 실패
- DB 잠김
- Import 실패
- 예상치 못한 런타임 예외

### 9.3 사용자 메시지 방향
- 가능한 짧고 단순한 문구 사용
- 예: "붙여넣기에 실패했습니다. 프로그램을 다시 시작합니다."
- 예: "이미지 붙여넣기에 실패했습니다. 파일 위치를 엽니다."

### 9.4 주의사항
프로그램을 죽였다가 다시 살리는 정책은 구현은 단순하지만, UX상 거칠고 데이터 손상 가능성을 높일 수 있다.  
따라서 실제 구현에서는 **가능한 경우 기능 단위 복구를 우선**하고, 복구 불가능한 상황에서만 프로세스 재시작을 사용한다.

## 11. 기술 요구사항

## 11.1 권장 스택
- Python 3.11 또는 3.12
- PySide6
- pywin32
- SQLite3
- Pillow
- PyInstaller
- Inno Setup

## 11.2 기술 방향
- GUI: PySide6
- 시스템 트레이: QSystemTrayIcon
- 글로벌 핫키: Windows RegisterHotKey 기반
- 마우스 트리거: pynput 또는 저수준 Win32 Hook
- 클립보드: Qt + Windows Clipboard API 보조
- 자동 붙여넣기: SendInput 계열
- 이미지 저장: 파일 시스템 assets 폴더
- 패키징: one-folder + 설치파일 방식

## 11.3 기술 메모
- 텍스트 붙여넣기는 필수 기능
- 이미지 붙여넣기는 대상 앱 호환성에 좌우됨
- 관리자 권한/UIPI 이슈는 사용자 요구에 따라 깊게 다루지 않지만, 구현 중 예외는 기록해야 함

## 12. 성능 및 안정성 요구

- 백그라운드 메모리: 80~150MB 목표
- 팝업 표시: 150ms 이내 목표
- 텍스트 붙여넣기: 300ms 이내 목표
- 이미지 붙여넣기: 1초 이내 목표
- 장시간 상주 중 비정상 메모리 증가 최소화
- 8시간 이상 실행 시 중대 오류 없이 동작 목표

메모리 정책:
- 전체 상용구를 한 번에 모두 메모리에 올리지 않음
- 본문은 필요 시 로딩
- 이미지 원본은 필요 시 로딩
- 썸네일 캐시는 제한
- 팝업 종료 시 이미지 참조 해제

## 17. 개발 도구 선정

### 선정 결과
**Cursor**

### 선정 이유
- 이 프로젝트는 단일 파일이 아니라 다수의 Python 파일, Qt UI, Win32 연동, 설치 스크립트, 문서 파일로 구성된다.
- 따라서 단순 코드 생성보다 **프로젝트 전체 구조를 보며 반복적으로 수정하는 IDE형 에이전트**가 유리하다.
- Cursor는 프로젝트 구조 생성, 다중 파일 편집, PRD 첨부 후 단계별 구현 지시에 적합하다.

### 주의
Cursor가 실제로 로컬 PC에서 GitHub 연동, 폴더 설정, 명령 실행을 자동으로 끝까지 “반드시 해준다”고 보장할 수는 없다.  
따라서 PRD에는 **Cursor에게 수행 지시를 내리는 방식**으로 문서를 제공한다.
